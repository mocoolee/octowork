# 門市 SOP 撰寫器

## 執行前
1. 讀 company.md → 營運標準、人事規範
2. 讀 products.md → 如果 SOP 涉及商品知識

## 輸入
使用者口述或文字描述一個流程：
- 「開店流程大概就是先開燈、開冷氣、點收備用金、檢查陳列、然後開門」
- 「客人進來的時候要先打招呼，然後觀察他看什麼...」

## 產出格式

```markdown
# [流程名稱] SOP

版本：v1.0
適用門市：全門市 / [特定門市]
最後更新：YYYY/MM/DD

## 目的
（一句話說明為什麼要有這個 SOP）

## 適用對象
（誰要照這個做）

## 流程步驟

### Step 1：[動作名稱]
- 做什麼：（具體描述）
- 怎麼做：（步驟拆解）
- 注意事項：（容易出錯的地方）
- 完成標準：（怎樣算做好了）

### Step 2：...

## 常見問題 Q&A
- Q：[門市人員可能會問的問題]
- A：[回答]

## 異常處理
- 如果遇到 [狀況]，則 [怎麼處理]
```

## 寫作原則
- 語言要白話到「高中生看得懂」的程度
- 每個步驟都要有「完成標準」，讓做的人知道做到什麼程度算 OK
- 加入「常見問題」，把你知道新人一定會問的問題先回答
- 流程超過 10 步驟就要分段，避免一大串
