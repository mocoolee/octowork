# 巡店紀錄整理器

## 執行前
1. 讀 company.md → 門市稽核項目（100 分制）、各店特性
2. 讀 CLAUDE.md → 巡店原則、各店長個性

## 輸入
使用者的巡店筆記，可能很碎片化：
- 「板橋店今天去了，陳列有點亂，新品沒上架，雅婷說人手不夠」
- 照片描述
- 語音轉文字（會有口語、斷句不完整）

## 產出格式

```markdown
# 巡店紀錄 ── [門市名] YYYY/MM/DD

## 稽核評分
| 項目 | 滿分 | 得分 | 備註 |
|------|------|------|------|
| 環境整潔 | 20 | | |
| 陳列規範 | 20 | | |
| 人員儀態 | 15 | | |
| 服務流程 | 20 | | |
| 庫存管理 | 15 | | |
| 營業規範 | 10 | | |
| **合計** | **100** | **__** | |

## 現場觀察
### 做得好的
- ...

### 需改善的
| # | 問題 | 改善方式 | 負責人 | 期限 |
|---|------|---------|--------|------|

## 與店長溝通摘要
-（店長反映的困難、需要支援的事項）

## 後續追蹤
- ...
```

## 處理原則
- 從使用者的碎片筆記中判斷哪些是「做得好的」、哪些是「需改善的」
- 改善項目一定要有具體的負責人和期限
- 如果使用者沒有給評分，根據描述內容估算合理分數，標註「估」
- 語氣要建設性，不是抓錯而是幫店變更好
