# 營運週報產生器

## 執行前
1. 讀 company.md → 各店目標、營運標準
2. 讀 CLAUDE.md → 陳經理的偏好（先結論後數據、一頁 A4、異常主動報）

## 輸入
使用者提供本週各店的彙整數據。可能是：
- 一整週的日報數字
- ERP 匯出的週彙整
- 口述重點

## 產出格式

```markdown
# A區營運週報 ── 第 N 週（MM/DD-MM/DD）

## 本週結論
（3-5 句話，講重點：整體達成狀況、最大亮點、最大隱憂、下週最重要的一件事）

## 業績總覽
| 門市 | 本週營收 | 週目標 | 達成率 | vs 上週 | 累計達成率 |
|------|---------|--------|--------|---------|-----------|

## 亮點
- [門市]：[具體事蹟]

## 需要關注
- [門市]：[問題] → [已採取行動 / 計畫行動]

## 重要事項
-（本週發生的客訴、人事異動、設備問題等）

## 下週計畫
| 項目 | 負責人 | 期限 |
|------|--------|------|
```

## 品質檢查
- [ ] 結論是不是一看就懂，不用再往下讀
- [ ] 每個數字有沒有比較基準（vs 上週 / vs 目標）
- [ ] 問題有沒有附上行動（不能只報問題不給方案）
- [ ] 長度有沒有控制在一頁 A4 以內
- [ ] 有沒有用到模糊語言（感覺、大概、應該）→ 刪掉
