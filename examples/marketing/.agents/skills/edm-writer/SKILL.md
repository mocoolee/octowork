# EDM 撰寫器

## 執行前
1. 讀 products.md → 要推的產品資訊、價格、賣點
2. 讀 company.md → 品牌語氣、客群 persona
3. 讀 CLAUDE.md → EDM 原則

## 輸入
使用者提供：
- 這封信要推什麼（產品 / 活動 / 內容）
- 目的（促銷轉換 / 內容經營 / 喚醒沉睡客 / 新品預告）
- 有沒有特別的促銷方案

## 產出格式

```markdown
# EDM ── [主題]

## 主旨行（3 組，選一個）
1. 「...」
2. 「...」
3. 「...」

## 預覽文字
（顯示在主旨行後面的那一小段，補充主旨沒說完的話）

## 信件內文

### 開場（2-3 句）
（從讀者的生活情境切入，不要一開頭就賣東西）

### 痛點（2-3 句）
（點出讀者可能有的困擾或需求）

### 產品介紹（3-5 句）
（怎麼解決上面的痛點，帶出產品特色）

### CTA
（明確的行動按鈕文字 + 連結說明）

### P.S.
（補充一個額外誘因：限時、贈品、或一句有溫度的話）
```

## 發信建議
- 建議發信時間：週二早上 10 點
- 建議分眾：根據內容建議適合的客群

## 品質檢查
- [ ] 主旨行 < 20 字
- [ ] 至少一個主旨行包含數字或好奇缺口（「你知道___嗎？」）
- [ ] 每封信只推 1-2 個產品，不要塞太多
- [ ] 開場從生活情境切入，不是「親愛的會員您好」
- [ ] 語氣像朋友聊天，不像廣告傳單
- [ ] CTA 只有一個主要行動（不要同時叫人買東西又叫人看文章）
