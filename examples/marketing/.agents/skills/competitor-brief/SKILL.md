# 競品快報

## 執行前
1. 讀 products.md → 我們的產品定價、賣點
2. 讀 company.md → 我們的品牌定位、客群、管道佈局
3. 讀 CLAUDE.md → 行銷原則

## 輸入
使用者提供：
- 競品名稱
- 想了解的面向（全面 / 定價 / 行銷策略 / 產品比較）
- 如果有競品的網站或社群連結更好

## 產出格式

```markdown
# 競品快報 ── [競品名稱] vs 日和生活

## 品牌概覽
| 項目 | 日和生活 | [競品] |
|------|---------|--------|
| 定位 | | |
| 主要客群 | | |
| 價格帶 | | |
| 主要通路 | | |

## 產品比較
| 品項 | 我們 | 對方 | 分析 |
|------|------|------|------|
| [對應品項] | 價格 / 特色 | 價格 / 特色 | 誰贏在哪 |

## 行銷策略觀察
- **廣告：** 他們在哪裡投、什麼調性
- **社群：** 發文頻率、互動風格、粉絲量級
- **促銷：** 常用什麼促銷手法
- **內容：** 有沒有做 SEO、部落格、KOL

## 他們做得好的
1. ...
2. ...

## 我們的機會點
1. （具體可以做的事，不是「要更努力」）
2. ...
3. ...
```

## 品質檢查
- [ ] 比較表有具體數字，不是「他們比較便宜」
- [ ] 「我們的機會點」是可執行的行動，不是口號
- [ ] 有分析不只有資訊（「他們這樣做，所以我們可以___」）
- [ ] 不要只看缺點，也要學對方做得好的地方
