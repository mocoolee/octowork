# 行銷週報產生器

## 執行前
1. 讀 company.md → 各管道現況、歷史數據
2. 讀 CLAUDE.md → David 的偏好（表格、先結論後數據、禁止模糊語言）

## 輸入
使用者貼上本週各管道的數據，可能來自：
- GA4 截圖或數字
- Google Ads / Meta Ads 後台數據
- MailerLite 的 EDM 數據
- Shopify 的營收數據
- 格式不固定，什麼格式都收

## 產出格式

```markdown
# 行銷週報 ── 第 N 週（MM/DD - MM/DD）

## 本週結論
（3 句話講完：整體表現、最大亮點、下週最重要的事）

## David 看這裡
| 指標 | 本週 | 上週 | 變化 | 月目標 | 進度 |
|------|------|------|------|--------|------|
| 營收 | | | | $500 萬 | |
| ROAS | | | | > 5 | |
| 新客佔比 | | | | 30% | |

## 各管道表現
| 管道 | 花費 | 營收 | ROAS | CPA | vs 上週 |
|------|------|------|------|-----|---------|
| Meta Ads | | | | | |
| Google Ads | | | | | |
| SEO | - | | ∞ | - | |
| EDM | - | | - | - | 開信率：__ |
| IG 導流 | - | | - | - | |

## 本週最佳
- **[素材/活動名稱]**：表現數據，為什麼好，能不能複製

## 本週最差
- **[素材/活動名稱]**：表現數據，為什麼差，已停 or 調整方向

## 異常說明
（如果有異常數據波動，主動解釋原因，不要等 David 問）

## 下週計畫
| 事項 | 優先級 | 預期效果 |
|------|--------|---------|
```

## 品質檢查
- [ ] 結論是不是一看就懂，不用再往下讀
- [ ] 所有數字都有 vs 上週的比較
- [ ] 沒有模糊語言（感覺、大概、應該、可能）
- [ ] David 的三個指標（營收、ROAS、新客比）放在最醒目的位置
- [ ] 異常要主動解釋，不等被問
- [ ] 最佳和最差都有分析「為什麼」，不只報數字
