# Landing Page 文案產生器

## 執行前
1. 讀 products.md → 產品賣點、價格、異議處理
2. 讀 company.md → 品牌語氣、客群 persona、歷史經驗
3. 讀 CLAUDE.md → 文案原則

## 輸入
使用者提供：
- 產品或活動名稱
- 目標客群（安心媽媽 / 質感獨居族 / 不指定）
- 促銷方案（如果有）
- 有沒有特別想強調的賣點

## 產出格式

```markdown
# LP 文案 ── [產品/活動名稱]

## Hero 區塊
- **大標：** （痛點＋解法結構，一句話講完核心價值）
- **副標：** （補充說明或數據佐證）
- **CTA 按鈕文字：**
- **視覺建議：** （建議用什麼圖/影片）

## 痛點區塊（為什麼你需要這個）
- 痛點 1：（從使用情境切入）
- 痛點 2：
- 痛點 3：

## 解法區塊（我們怎麼幫你）
（產品如何解決上面的痛點，先情境再規格）

## 產品特色（3-4 個）
每個特色：icon 建議 + 標題 + 一句話說明

## 社會證明
- 數據佐證（銷售量、回購率、評分）
- 顧客評價 2-3 則（從真實評論改寫）
- 檢測報告（SGS 等）

## FAQ（3-5 題）
挑最常被問的問題，處理購買前的疑慮

## 最終 CTA
- **標題：** （急迫感或價值重申）
- **按鈕文字：**
- **安心保證：** （退換貨政策、付款安全）
```

## 品質檢查
- [ ] Hero 大標用「痛點＋解法」結構
- [ ] 所有文案先寫情境、再寫規格
- [ ] FAQ 有處理 products.md 裡的「異議」
- [ ] CTA 按鈕文字具體（「立即體驗 $399」比「了解更多」好）
- [ ] 語氣溫暖真誠，不像購物台
- [ ] 段落短、重點粗體，手機閱讀友善
