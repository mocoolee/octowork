# OctoWork 的運作原理

## 為什麼叫 OctoWork？

章魚有八隻手臂，每隻都能獨立做事。大腦負責決策，手臂負責執行。

你的 work-os 也是這樣：CLAUDE.md 是大腦（決策和原則），Skills 是八隻手臂（各自執行不同任務），Workflows 是手臂之間的協調（按順序完成一整套工作）。

一個人，八隻手。一個人就是一個部門。

## 系統的三層結構

### 第一層：背景檔（你是誰）

三個 Markdown 檔案，AI 每次開機都會讀：

| 檔案 | 內容 | 誰維護 |
|------|------|--------|
| CLAUDE.md | 你的角色、KPI、做事原則、協作者 | 你自己 |
| company.md | 公司背景、品牌、客群、標準 | 你或主管 |
| products.md | 產品/服務資料 | 視需要 |

### 第二層：Skills（你會做什麼）

每個 Skill 是一個資料夾，裡面有一個 SKILL.md 定義「這件事怎麼做」。

Skill 不是你事先規劃的，而是做中長出來的——你做了第二次同樣的事，就跟 AI 說「把剛才的做法存成 Skill」。

### 第三層：Workflows（你的流水線）

把多個 Skill 串成一個流程。一句話觸發，AI 按步驟跑完。

## 越做越聰明的機制

每次做完一件事，AI 會問你：「有沒有什麼值得記下來的？」

你說的經驗會被加到 CLAUDE.md 的「做事原則」裡。下次做事，AI 會參考這些原則。

你用得越久，做事原則越厚，AI 越像你。

## 升級不影響你的資料

OctoWork 把「框架」和「你的資料」完全分開。框架在 core/、examples/、docs/，你的資料在 workspace/。升級時只動框架，不動你的資料。
