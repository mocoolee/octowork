# 怎麼寫好的 Workflow

## 什麼時候該建 Workflow

當你有一串工作是「做完 A 就要做 B 再做 C」，而且順序差不多固定，就該串成 Workflow。

通常一個 Workflow 會串 2-5 個 Skill。

## Workflow 檔案的結構

放在 `workflows/` 資料夾，每個流程一個 .md 檔。

```markdown
# 流程名稱

## 觸發
我說「...」時啟動這個流程

## 步驟

### 1. 步驟名稱
- 做什麼
- 用哪個 Skill（如果有）
- 需不需要停下來讓我確認

### 2. 步驟名稱
- ...

## 輸出
- 存到哪裡
- 有沒有精簡版（給 LINE、給主管）
```

## 三個設計原則

### 1. 在關鍵節點加「人工確認」

不要讓 AI 一路跑到底。在重要的輸出點加「停下來給我確認」。特別是：涉及對外發送的內容、涉及金額判斷的決策、涉及人員安排的排班。

### 2. 輸出要有「可直接發出去」的版本

你的 Workflow 產出不只是存在電腦裡的檔案。通常你需要一個「可以直接貼到 LINE 群」或「可以直接 email 給主管」的精簡版。在 Workflow 裡就要定義這個。

### 3. 最後一步問「要不要更新做事原則」

這是讓系統持續進化的機制。跑完流程之後，養成反思的習慣。
