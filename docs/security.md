# 資料安全與隱私

用 OctoWork 之前，花兩分鐘了解一下你的資料會怎麼被處理。

---

## 你的檔案在哪裡

所有 OctoWork 的 Markdown 檔案都存在你自己的電腦上（`workspace/` 資料夾）。

- `workspace/` 被 `.gitignore` 保護，不會被 `git push` 到任何地方
- 框架升級（`git pull`）也不會動到你的 `workspace/`
- 你的 CLAUDE.md、company.md、products.md 只存在你的電腦裡

---

## 但是：AI 會讀到你的內容

當你使用 Claude Code 或 Gemini CLI 時，你跟 AI 對話的過程中，相關的檔案內容（CLAUDE.md、company.md、products.md、Skills 等）會被傳送到 AI 服務的伺服器進行處理。

這跟你用 ChatGPT 或任何雲端 AI 是一樣的——你貼進去的東西，AI 服務商的伺服器會處理。

各服務商的資料政策：
- **Claude Code（Anthropic）**：[隱私政策](https://www.anthropic.com/policies/privacy)
- **Gemini CLI（Google）**：[服務條款](https://ai.google.dev/terms)

---

## 什麼資料不該放進來

以下類型的資料，建議不要寫進 company.md 或 products.md：

- 密碼、API key、存取金鑰
- 身分證字號、信用卡號、銀行帳號等個人識別資料
- 公司明確標示為「機密」或「限閱」的文件內容
- 客戶的完整個人資料（用代號或去識別化處理）
- 未公開的財務數據（營收、利潤等精確數字）

---

## 那我的 company.md 該寫什麼？

寫「對外可以說的」那些資訊就好：

- **公司簡介、品牌定位、產品特色** → 這些本來就是公開的
- **客群描述** → 用 persona，不要用真實客戶資料
- **做事的流程和原則** → 這是方法論，不是機密
- **價格帶和折扣規則** → 如果不是商業機密的話

簡單來說：想像這份檔案如果被別人看到，你不會緊張的內容，就可以放。

---

## 如果你的公司有資安規定

- 使用前先確認公司是否允許使用外部 AI 工具
- 醫療、金融、法律等受監管產業要特別注意合規要求
- 如果不確定，問你的 IT 部門或主管最保險

---

## 實務建議

1. **先從不敏感的工作開始用**，例如寫報告格式、整理 SOP、規劃排程
2. **需要處理客戶資料時，先去識別化**（用代號取代真名、模糊化數字）
3. **定期檢查 CLAUDE.md 和 company.md**，確認沒有不小心加入敏感資訊
4. **做事原則裡提到的人名**，考慮只用職稱或代號（「主管」比「陳經理」安全）
